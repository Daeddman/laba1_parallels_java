# ✅ Валідація введення даних

## 🎯 Що додано

### 1. Обмеження кількості потоків
- **Максимум:** 32 потоки
- **Мінімум:** 1 потік
- Константа: `MAX_THREADS = 32`

### 2. Перевірка введення кількості потоків
```java
while (true) {
    System.out.print("Введіть кількість потоків (1-32): ");
    try {
        threadCount = scanner.nextInt();
        if (threadCount > 0 && threadCount <= MAX_THREADS) {
            break;
        } else {
            System.out.println("❌ Помилка: кількість потоків має бути від 1 до 32!");
        }
    } catch (Exception e) {
        System.out.println("❌ Помилка: введіть ціле число!");
        scanner.nextLine(); // Очистити буфер
    }
}
```

### 3. Перевірка кроку послідовності (для кожного потоку)

## 🔒 Переваги валідації

### 1. Захист від помилок
- Неможливо ввести некоректні дані
- Програма не "впаде" через невірні значення
- Користувач отримує зрозумілі повідомлення про помилки

### 2. Обмеження ресурсів
- Максимум 32 потоки запобігає перевантаженню системи
- Захист від випадкового введення великих чисел (наприклад, 1000 потоків)

### 3. Логічність даних
- Крок та час завжди позитивні
- Неможливі нелогічні значення (0 або від'ємні)

### 4. Зручність користувача
- Чіткі підказки в дужках: `(1-32)`, `(>0)`
- Емодзі для наочності: ❌ помилка, ✅ успіх
- Повторний запит при помилці (не треба перезапускати програму)

## 💡 Технічні деталі

### Try-Catch блоки
```java
try {
    value = scanner.nextInt();
    // Перевірка значення
} catch (Exception e) {
    System.out.println("❌ Помилка: введіть ціле число!");
    scanner.nextLine(); // ВАЖЛИВО: очистити буфер!
}
```

**Чому `scanner.nextLine()`?**
- Після помилки nextInt() залишає символ нового рядка в буфері
- nextLine() очищує буфер
- Без цього цикл буде нескінченним при введенні тексту

### Цикл while(true)
```java
while (true) {
    // Спроба введення
    if (умова_коректності) {
        break; // Вихід з циклу при коректному введенні
    }
    // Інакше - повторити
}
```
