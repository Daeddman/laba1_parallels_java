# Інструкція для виконання лабораторної роботи

## Тема: Засоби створення багатопоточних програм

---

## Кроки виконання

### Крок 1: Компіляція програми

```bash
cd src
javac Main.java
```

### Крок 2: Запуск програми вручну

```bash
java Main
```

Після запуску введіть параметри:
- Кількість потоків (наприклад: 3)
- Крок послідовності (наприклад: 5)
- Затримка зупинки в мілісекундах (наприклад: 1000)

### Крок 3: Автоматичне тестування

Для автоматичного тестування з різними параметрами:

```bash
./run_tests.sh
```

## Варіанти тестування для звіту

### Варіант 1: 2 потоки
```
Кількість потоків: 2
Крок послідовності: 10
Затримка зупинки: 500 мс
```

**Що очікувати:**
- 2 робочі потоки обчислюють суми
- Перший потік зупиняється через 500 мс
- Другий потік зупиняється ще через 500 мс
- Кожен потік виведе свій результат

### Варіант 2: 4 потоки
```
Кількість потоків: 4
Крок послідовності: 5
Затримка зупинки: 1000 мс
```

**Що очікувати:**
- 4 робочі потоки працюють паралельно
- Кожен наступний потік зупиняється на 1 секунду пізніше
- Загальний час виконання: ~4 секунди

### Варіант 3: 8 потоків
```
Кількість потоків: 8
Крок послідовності: 3
Затримка зупинки: 1500 мс
```

**Що очікувати:**
- Максимальне навантаження на процесор
- 8 паралельних потоків
- Добре видно завантаження ядер процесора
- Загальний час: ~12 секунд

## Створення скріншотів для звіту

### 1. Скріншоти виконання програми

Для кожного варіанту треба зробити скріншот виведення програми, де видно:
- Введені параметри
- Запуск потоків
- Зупинку потоків
- Результати роботи кожного потоку (номер, сума, кількість елементів)

### 2. Скріншоти диспетчера задач (Activity Monitor на macOS)

**Як відкрити Activity Monitor:**
1. Натисніть `Cmd + Space`
2. Введіть "Activity Monitor"
3. Відкрийте додаток

**Що показати на скріншоті:**
1. Перейдіть на вкладку "CPU"
2. Знайдіть процес "java"
3. Внизу подивіться на графіки завантаження процесора
4. Зробіть скріншот під час виконання програми

**Для кожного варіанту:**
- Запустіть програму з певною кількістю потоків
- Швидко переключіться на Activity Monitor
- Зробіть скріншот, поки потоки працюють
- Зверніть увагу на завантаження ядер CPU

**Альтернатива - термінальна утиліта:**
```bash
# Запустіть в окремому терміналі перед тестуванням
top -pid $(pgrep java)
```

## Підготовка звіту

### Структура звіту

1. **Титульна сторінка**
   - Назва лабораторної роботи
   - Тема: Засоби створення багатопоточних програм
   - ПІБ студента
   - Група
   - Дата виконання

2. **Мета роботи**
   - Познайомитись із засобами побудови паралельних програм різними мовами програмування

3. **Теоретичні відомості**
   - Коротко про багатопоточність
   - Засоби Java для багатопоточності
   - Опис класів Thread, використаних у роботі

4. **Опис програми**
   - Архітектура рішення
   - Клас SumThread - для обчислення сум
   - Клас ControllerThread - для керування зупинкою
   - Головний клас Main

5. **Код програми**
   - Повний лістинг Main.java з коментарями

6. **Результати тестування**
   
   **Для кожного варіанту (2, 4, 8 потоків):**
   - Таблиця параметрів:
     ```
     | Параметр | Значення |
     |----------|----------|
     | Кількість потоків | X |
     | Крок послідовності | Y |
     | Затримка зупинки | Z мс |
     ```
   - Скріншот виконання програми
   - Скріншот Activity Monitor з завантаженням CPU
   - Аналіз результатів:
     * Які значення сум отримали різні потоки
     * Чому результати відрізняються
     * Як завантажувався процесор

7. **Порівняльний аналіз**
   - Таблиця результатів для різної кількості потоків
   - Висновки про вплив кількості потоків на результати
   - Спостереження за завантаженням CPU

8. **Відповіді на контрольні запитання**
   - Використайте файл CONTROL_QUESTIONS.md

9. **Посилання на Git-репозиторій**
   - URL репозиторію з кодом
   - Інструкція для клонування та запуску

10. **Висновки**
    - Що навчились робити
    - Які особливості багатопоточності виявили
    - Переваги та недоліки використаних підходів

## Поради для успішного виконання

1. **Тестуйте з різною кількістю потоків** - від 1 до кількості ядер процесора × 2

2. **Робіть скріншоти під час виконання** - не після завершення

3. **Експериментуйте з параметрами:**
   - Малі затримки (100-500 мс) - потоки швидко зупиняються
   - Великі затримки (2000-5000 мс) - можна побачити більші суми

4. **Спостерігайте за CPU:**
   - При малій кількості потоків - неповне використання
   - При великій кількості - максимальне навантаження
   - Коли потоків більше ніж ядер - перемикання контексту

5. **Аналізуйте результати:**
   - Чому суми в різних потоках відрізняються?
   - Як час роботи впливає на результат?
   - Чи справедливо розподіляється час між потоками?

## Приклад аналізу результатів

```
Тест з 4 потоками, крок=5, затримка=1000мс:

Потік №1: Сума = 2500, Кількість елементів = 100
Потік №2: Сума = 5000, Кількість елементів = 200  
Потік №3: Сума = 7500, Кількість елементів = 300
Потік №4: Сума = 10000, Кількість елементів = 400

Аналіз:
- Кожен наступний потік працював на 1 секунду довше
- Кількість елементів приблизно зростає лінійно
- Сума також зростає, оскільки S = n*(n-1)*step/2
- CPU завантажений на ~80% на 4-ядерному процесорі
```
