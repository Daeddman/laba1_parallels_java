## 📊 Аналіз результатів

### Спостереження:

1. **Лінійне зростання часу роботи**
   - Кожен наступний потік працює довше на величину затримки
   - Потік №N працює приблизно N × затримка мс

2. **Кількість обчислених елементів**
   - Приблизно пропорційна часу роботи
   - Швидкість: ~700-1000 елементів/секунду на потік

3. **Обчислені суми**
   - Зростають квадратично з кількістю елементів
   - Формула: S = step × n × (n-1) / 2

4. **Паралельність**
   - Всі потоки працюють одночасно
   - Незалежні обчислення в кожному потоці
   - Коректна зупинка через volatile прапорець

## ⚠️ Важливі нотатки

1. **Volatile:**
   - Використовується для `running` прапорця
   - Гарантує видимість змін між потоками
   - Не потребує синхронізації для простих операцій read/write

2. **Thread.sleep():**
   - Може кидати `InterruptedException`
   - Не гарантує точного часу затримки
   - Використовується для демонстрації, не для точного таймінгу

3. **Deprecated методи:**
   - ❌ Не використовувати `Thread.stop()`
   - ✅ Використовувати прапорець для зупинки

4. **Join:**
   - Очікує завершення потоку
   - Необхідний для коректного завершення програми
